# 用户同步逻辑重构计划（修订版）

## 1. 问题分析

### 当前重复代码

**位置 1**: `internal/downloading/features.go`
- `syncUser()` - 更新用户数据库记录
- `syncUserAndEntity()` - 同步用户和实体

**位置 2**: `internal/profile/downloader.go`
- `syncUserDirectory()` - 重新实现了相同的逻辑

### 原方案的问题

原计划让 `profile` 包依赖 `downloading` 包，这**违背了单一职责原则**：
- `downloading` 包的职责是**推文下载**
- `profile` 包的职责是**用户资料下载**
- 用户同步逻辑不属于"推文下载"的职责范畴

## 2. 重构方案：创建独立的 `internal/user` 包

### 模块职责划分

| 模块 | 职责 | 依赖 |
|------|------|------|
| `internal/user` | 用户数据同步、目录管理（新建） | `database`, `naming`, `utils` |
| `internal/downloading` | 推文下载（调用 `user` 包） | `user`, `twitter`, `database` |
| `internal/profile` | 用户资料下载（调用 `user` 包） | `user`, `database`, `downloader` |
| `internal/downloading/entity.go` | 实体 CRUD 操作（保持不变） | `database` |

### 依赖关系图

```
                    ┌─────────────┐
                    │  internal/  │
                    │    user     │ ← 新建：用户同步核心逻辑
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
           ▼               ▼               ▼
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │ downloading │ │   profile   │ │  (其他模块)  │
    │  (推文下载)  │ │ (资料下载)  │ │             │
    └─────────────┘ └─────────────┘ └─────────────┘
```

## 3. 详细修改步骤

### 步骤 1: 创建 `internal/user` 包

**新文件**: `internal/user/sync.go`

```go
// Package user 提供用户数据同步和目录管理的核心功能
package user

import (
	"fmt"
	"os"
	"path/filepath"

	"github.com/jmoiron/sqlx"
	"github.com/unkmonster/tmd/internal/database"
	"github.com/unkmonster/tmd/internal/naming"
)

// SyncUserData 同步用户数据到数据库
// 返回值: isNew (是否新用户), renamed (是否改名), error
func SyncUserData(db *sqlx.DB, userID uint64, name, screenName string, isProtected bool, friendsCount int) (isNew bool, renamed bool, err error) {
	usrdb, err := database.GetUserById(db, userID)
	if err != nil {
		return false, false, err
	}

	isNew = usrdb == nil
	renamed = false

	if isNew {
		usrdb = &database.User{}
		usrdb.Id = userID
	} else {
		renamed = usrdb.Name != name || usrdb.ScreenName != screenName
	}

	usrdb.FriendsCount = friendsCount
	usrdb.IsProtected = isProtected
	usrdb.Name = name
	usrdb.ScreenName = screenName

	if isNew {
		err = database.CreateUser(db, usrdb)
	} else {
		err = database.UpdateUser(db, usrdb)
	}
	if err != nil {
		return false, false, err
	}

	if renamed || isNew {
		err = database.RecordUserPreviousName(db, userID, name, screenName)
	}

	return isNew, renamed, err
}

// EntityRef 表示一个用户实体的引用
type EntityRef struct {
	ID        int
	UserID    uint64
	Name      string
	ParentDir string
}

// SyncUserDirectory 同步用户目录
// 返回用户目录路径和实体引用
func SyncUserDirectory(db *sqlx.DB, userID uint64, name, screenName, parentDir string, expectedTitle ...string) (*EntityRef, error) {
	// 同步用户数据
	_, _, err := SyncUserData(db, userID, name, screenName, false, 0)
	if err != nil {
		return nil, err
	}

	// 计算期望的目录名
	var title string
	if len(expectedTitle) > 0 && expectedTitle[0] != "" {
		title = expectedTitle[0]
	} else {
		userNaming := naming.NewUserNaming(name, screenName)
		title = userNaming.SanitizedTitle()
	}

	// 查找或创建实体
	entity, err := database.LocateUserEntity(db, userID, parentDir)
	if err != nil {
		return nil, err
	}

	if entity == nil {
		// 创建新实体
		entity = &database.UserEntity{
			Uid:       userID,
			ParentDir: parentDir,
			Name:      title,
		}
		userDir := filepath.Join(parentDir, title)
		if err := os.MkdirAll(userDir, 0755); err != nil {
			return nil, err
		}
		if err := database.CreateUserEntity(db, entity); err != nil {
			return nil, err
		}
		return &EntityRef{
			ID:        int(entity.Id.Int32),
			UserID:    userID,
			Name:      title,
			ParentDir: parentDir,
		}, nil
	}

	// 检查是否需要重命名
	oldUserDir := entity.Path()
	if entity.Name == title {
		if err := os.MkdirAll(oldUserDir, 0755); err != nil && !os.IsExist(err) {
			return nil, err
		}
		return &EntityRef{
			ID:        int(entity.Id.Int32),
			UserID:    userID,
			Name:      title,
			ParentDir: parentDir,
		}, nil
	}

	// 重命名目录
	newUserDir := filepath.Join(parentDir, title)
	if err := os.Rename(oldUserDir, newUserDir); err != nil {
		if os.IsNotExist(err) {
			if mkdirErr := os.MkdirAll(newUserDir, 0755); mkdirErr != nil {
				return nil, mkdirErr
			}
		} else {
			return nil, err
		}
	}

	// 更新数据库
	entity.Name = title
	if err := database.UpdateUserEntity(db, entity); err != nil {
		return nil, err
	}

	return &EntityRef{
		ID:        int(entity.Id.Int32),
		UserID:    userID,
		Name:      title,
		ParentDir: parentDir,
	}, nil
}

// Path 返回用户目录的完整路径
func (e *EntityRef) Path() string {
	return filepath.Join(e.ParentDir, e.Name)
}
```

// syncUser 同步用户数据（内部使用 user 包）
func syncUser(db *sqlx.DB, user *twitter.User) error {
	_, _, err := userpkg.SyncUserData(db, user.Id, user.Name, user.ScreenName, user.IsProtected, user.FriendsCount)
	return err
}

// syncUserAndEntity 同步用户和实体
func syncUserAndEntity(db *sqlx.DB, user *twitter.User, dir string) (*UserEntity, error) {
	// 使用 user 包同步目录
	ref, err := userpkg.SyncUserDirectory(db, user.Id, user.Name, user.ScreenName, dir)
	if err != nil {
		return nil, err
	}

	// 转换为 UserEntity（保持向后兼容）
	entity, err := NewUserEntity(db, user.Id, dir)
	if err != nil {
		return nil, err
	}
	
	return entity, nil
}

## 4. 需要考虑的问题

### 问题 1: `UserEntity` 的额外方法

`downloading/entity.go` 中的 `UserEntity` 有额外方法：
- `LatestReleaseTime()` / `SetLatestReleaseTime()`
- `ClearLatestReleaseTime()`
- `Uid()`

**解决方案**: 保持 `UserEntity` 不变，`syncUserAndEntity` 仍然返回 `*UserEntity`，但内部调用 `user.SyncUserDirectory`。

### 问题 2: `syncPath` 函数

`syncPath` 是 `downloading/entity.go` 中的核心函数，用于 `UserEntity` 和 `ListEntity`。

**解决方案**: 
- `user` 包的 `SyncUserDirectory` 自己实现目录同步逻辑
- `downloading` 包的 `syncPath` 继续用于 `ListEntity`
- 或者把 `syncPath` 的逻辑也提取到 `user` 包（但 `ListEntity` 不是用户实体）

### 问题 3: 符号链接更新

`updateUserLink` 函数在 `downloading/entity.go` 中，用于更新列表中的用户符号链接。

**解决方案**: 这个函数与推文下载的列表功能紧密相关，保持在 `downloading` 包中。
